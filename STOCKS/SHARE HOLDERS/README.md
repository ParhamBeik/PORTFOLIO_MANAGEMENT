# Shareholders Data Pipeline

## 1. Purpose
This folder implements a historical shareholder-change fetcher for TSETMC symbols. It mirrors the same operational model used by Transactions: ledger-based progress tracking, reputation-aware symbol skipping, helper symbol lists, and detailed reporting.

## 2. Architecture
The pipeline is split into:
- Input universe and controls in `data/inputs/`.
- Runtime status in `data/tracking/` and `data/reputation/`.
- Fetched JSON output in `data/FETCH_SHAREHOLDER/` organized by industry and symbol.

Core scripts:
- `scripts/build_shareholder_tracking.py`: creates/extends date ledger and backfills success from existing files.
- `scripts/fetch_tsetmc_shareholders.py`: main fetch runner, reputation/list updates, cleanup.
- `scripts/generate_shareholder_status_report.py`: visual status analytics output (PNG charts).

## 3. Directory Layout
- `data/inputs/`
- `stocks_data.csv`: master symbol universe.
- `market_holidays.json`: non-trading dates.
- `filtered_stocks_data.csv`: symbols with successful shareholder payloads.
- `http_banned_stocks_data.csv`: symbols currently HTTP-banned by reputation.
- `timeout_stocks_data.csv`: symbols currently marked timeout-prone.
- `data/tracking/`
- `shareholder_tracking.csv`: ledger matrix, rows=symbols, columns=trading dates.
- `data/reputation/`
- `shareholder_reputation.json`: per-symbol counters, ban flags, history.
- `data/FETCH_SHAREHOLDER/`
- `{industry}/{symbol}/{YYYY-MM-DD}.json`
- `reports/`
- `shareholder_report_overview.png`
- `shareholder_report_timeline.png`
- `shareholder_reputation_report.json`
- `scripts/`
- `build_shareholder_tracking.py`
- `fetch_tsetmc_shareholders.py`
- `generate_shareholder_status_report.py`

## 4. Tracking State Codes
Ledger values in `shareholder_tracking.csv`:
- `1`: success
- `2`: empty (valid API response but no usable shareholder sections)
- `3`: timeout
- `4`: not fetched
- `5`: HTTP error
- `6`: skipped due to bad HTTP reputation
- `7`: skipped due to timeout reputation

## 5. Runtime Flow
1. Load symbols, holidays, reputation, and helper CSV lists.
2. Read trading-date columns from `shareholder_tracking.csv`.
3. Process dates from newest to oldest.
4. Per date, collect symbols in states `4/3/5`.
5. Apply optional static helper filtering (filtered list / HTTP-banned list / timeout list).
6. Apply runtime reputation skip (state `6`/`7`) when reputation filter is enabled.
7. Fetch payload, persist JSON for successes, update ledger state.
8. Update reputation counters and helper lists.
9. Remove empty symbol folders.
10. Write reputation summary report.

## 6. Symbol Universe and Filtering
Active universe starts from `stocks_data.csv` then filtered by:
- Optional filtered-only mode via `USE_FILTERED_SYMBOLS_IF_AVAILABLE`.
- Optional HTTP-banned exclusion via `USE_HTTP_BANNED_STOCKS_LIST_FILTER`.
- Optional timeout exclusion via `USE_TIMEOUT_STOCKS_LIST_FILTER`.

Runtime (reputation) skipping is controlled separately by `USE_REPUTATION_FILTER`.

Practical meaning:
- `stocks_data.csv` is canonical.
- Filtered/bad/timeout CSV files are assistive optimization layers.

## 7. Reputation and Ban Logic
Reputation file: `data/reputation/shareholder_reputation.json`.

Main thresholds/controls in `scripts/fetch_tsetmc_shareholders.py`:
- `MAX_TOTAL_HTTP_ERRORS`
- `MAX_CONSECUTIVE_TIMEOUTS`
- `MAX_DAILY_ATTEMPTS`

Behavior summary:
- HTTP error accumulation can trigger HTTP ban.
- Consecutive timeout accumulation can trigger timeout ban.
- Successful responses clear timeout streak.
- Banned symbols are skipped and recorded in states `6`/`7`.

## 8. Date and Market-Time Logic
- Date columns are generated by `build_shareholder_tracking.py` from `TRACKING_START_DATE`.
- Today is included only after market close (`MARKET_CLOSE_HOUR`, `MARKET_CLOSE_MINUTE`).
- Fetch runner consumes existing columns newest-first.
- Holiday dates in `market_holidays.json` are skipped.
- A short pacing delay `REQUEST_DELAY` is applied between symbol calls.

## 9. Output JSON Contract
Each file in `data/FETCH_SHAREHOLDER/{industry}/{symbol}/{date}.json` contains:
- `shareholder_data`: API sections (typically before/after trade holders).
- `meta`: symbol, industry, fetch timestamp, date, and section count.

## 10. Reports
- `shareholder_report_overview.png`: high-level state, completion, and helper-set view.
- `shareholder_report_timeline.png`: recent completion/pending/error trend.
- `shareholder_reputation_report.json`: reputation totals, list totals, and API usage.

## 11. Script Responsibilities
- `scripts/build_shareholder_tracking.py`
- Initializes ledger if missing.
- Adds newly eligible dates as columns.
- Marks success for already-existing JSON files.
- Cleans empty folders.
- `scripts/fetch_tsetmc_shareholders.py`
- Main fetch loop with progress output.
- Updates ledger + reputation + helper CSV lists.
- Prunes empty symbol directories.
- `scripts/generate_shareholder_status_report.py`
- Builds visual status reports from the ledger and removes legacy text/json status reports.

## 13. Main Configuration Flags
In `scripts/fetch_tsetmc_shareholders.py`:
- API/runtime: `API_DAILY_LIMIT`, timeout values, `REQUEST_DELAY`.
- Reputation: `MAX_TOTAL_HTTP_ERRORS`, `MAX_CONSECUTIVE_TIMEOUTS`, `MAX_DAILY_ATTEMPTS`, `USE_REPUTATION_FILTER`.
- Universe filter controls: `USE_FILTERED_SYMBOLS_IF_AVAILABLE`, `USE_HTTP_BANNED_STOCKS_LIST_FILTER`, `USE_TIMEOUT_STOCKS_LIST_FILTER`.

## 14. Recovery and Maintenance
- Rebuild ledger + backfill from files via `build_shareholder_tracking.py`.
- Existing valid files are detected and avoid duplicate writes.
- Empty symbol folders are removed automatically.

## 15. Troubleshooting
- `Tracking CSV not found`: run `build_shareholder_tracking.py` first.
- Missing date column in fetch: rebuild tracking to regenerate date columns.
- Unexpected folder placement: verify industry/symbol mapping in input CSV and fetch path logic.
- Too many symbols skipped: inspect `shareholder_reputation.json` and helper CSV filters.
